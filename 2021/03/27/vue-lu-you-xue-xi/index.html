Vue路由学习

发表于2021-03-27|更新于2021-03-28
|阅读量:

路由学习，下周就要用，赶紧冲一把

  • 下载与安装：路由官网

  • 至于在Vue3.0版本中如何使用，这个使用就要参考element-ui的引入方式了嗷！！！Element-UI中，对于外界包的引入有指导作用，想不起来可以去找那个嗷！！！

  • 等等我发现这个vue add router无敌！！！直接帮你自动干好一切工作，绝了，👍！！！

  • components里面主要是公有组件，单页面主要放在view里面嗷！！！

  • scss的要求比较严格，这个时候在底下的style里面如果写对应的样式的话，.就只能够用于访问class类型的标签了嗷。

路由设置

  • router.js
  • 由于本身设置好了，你可以照着那个样式来写router规则即可。
  • App.vue中，可以添加导航栏
<div id="nav">
    <router-link to="/">Home</router-link>
    <router-link to="/about">About</router-link>
</div>
<router-view></router-view> //导航跳转的页面会加载到这里来

  • 进入页面之后直接由路由接管跳转，他会在你注册的路由里面去寻找响应的子组件并且渲染到合适的位置上返回页面给你，如果

    这个没有被设置，那么默认返回给你的就是一个空页面localhost:8080/#/

  • 控制登录的话就需要路由守卫啦！！！

  • 页面输入的name1，从动态路由的名字到组件里面来。
  • 编程式导航则是通过点击下面的，将信息传到上面的输入栏来。

嵌套路由：

  • 渲染位置：

  • 感觉和那个差不多？？？路由嵌套组件会渲染到父组件的这个位置。
  • $router.params.id这样式可以获取到上面那个位置填入的id的嗷。

路由重定向：

  • *代表输入的匹配不上嗷
  • 重定向关键词：redirect
  • /iview_router/:name，这个时候不写name的话，也会重定向到这里嗷
  • 名字在命名路由中使用嗷

路由参数传递：

  1. 就上面讲的/:id这样可以拿到参数

  2. 和父穿子的方法和做法是一样的嗷

  3. 如果还要往子组件传的话，再用父传子，或者bus也可嗷。

路由守卫：

全局守卫：

router.beforeEach(to,from,next)=>{...}全局前置守卫

router.beforeEach(to,from,next){
	if(to.path == '/iview_index'){
		if(auth.islogin()){
			next()
		}else{
			next('/iview_login')
		}
	}else{
        next()
    }
}
  • islogin()实际过程中要后端传数据进行验证的嗷

局部守卫：

  • 新建组件之前，先进行守卫，判断是否应该新建这样一个组件。
  • 如果要时时刻刻更新就要用watch了，因为组件那边，不重新加载页面时不会重新创建这个组件的，只会重载页面。
